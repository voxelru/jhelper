# jhelper

Flask-приложение для планирования задач из Jira: выборка по JQL, диаграмма «сотрудник × дата» с прямоугольниками трудозатрат. **Только чтение** — в Jira ничего не записывается.

## Возможности

- Подключение к Jira (Server / Data Center) по **логину и паролю** (HTTP Basic Auth)
- Выбор задач по **JQL** из файла настроек
- Диаграмма:
  - **сверху** — даты (выходные на шкале не занимают место)
  - **слева** — сотрудники
  - **прямоугольники** — задачи, ширина = плановые трудозатраты в рабочих днях
  - цвет по **приоритету**
  - внутри строки задачи идут **подряд без зазоров**, сначала более приоритетные
- Перетаскивание задач между сотрудниками и по горизонтали (**только в UI**, без синхронизации с Jira)
- Горизонт планирования настраивается (по умолчанию 90 календарных дней)
- Задачи без исполнителя **не показываются**

## Требования

- Python 3.10+
- Доступ к Jira REST API (`/rest/api/2/search` или `/rest/api/3/search`)
- Учётная запись с правом **чтения** задач по вашему JQL

## Настройка

### 1. Учётные данные Jira

Скопируйте `.env.example` в `.env` и заполните:

```env
JIRA_BASE_URL=https://jira.example.com
JIRA_USERNAME=login
JIRA_PASSWORD=secret
```

| Переменная       | Описание                          |
|------------------|-----------------------------------|
| `JIRA_BASE_URL`  | Базовый URL Jira без `/` в конце  |
| `JIRA_USERNAME`  | Логин                             |
| `JIRA_PASSWORD`  | Пароль                            |

Файл `.env` в git не коммитится.

### 2. Бизнес-настройки — `config/app_settings.yaml`

#### JQL

```yaml
jql: 'assignee is not EMPTY AND statusCategory != Done ORDER BY priority DESC'
```

Рекомендуется оставлять условие `assignee is not EMPTY`: задачи без исполнителя всё равно отфильтровываются на стороне приложения.

#### Приоритеты и цвета

Имена должны совпадать с `priority.name` в вашей Jira:

```yaml
priorities:
  order:
    - Highest
    - High
    - Medium
    - Low
    - Lowest
  colors:
    Highest: "#c62828"
    High: "#ef6c00"
    Medium: "#f9a825"
    Low: "#2e7d32"
    Lowest: "#546e7a"
    default: "#78909c"
```

- `order` — порядок в очереди сотрудника (первый = самый высокий приоритет)
- `colors` — заливка прямоугольников; неизвестный приоритет берёт `default`

#### Горизонт планирования

```yaml
planning_horizon:
  kind: calendar_days   # или working_days
  count: 90
```

| `kind`           | Смысл |
|------------------|--------|
| `calendar_days`  | `count` календарных дней от сегодня; на шкале только рабочие дни внутри интервала |
| `working_days`   | `count` рабочих дней подряд; выходные не занимают место на шкале |

#### Трудозатраты (ширина блока)

```yaml
effort:
  type: timetracking_original
  jira_field_id: null

min_effort_working_days: 0.25
working_hours_per_day: 8
```

| `effort.type`             | Источник |
|---------------------------|----------|
| `timetracking_original`   | `timetracking.originalEstimateSeconds` → дни через `working_hours_per_day` |
| `timetracking_remaining`  | `timetracking.remainingEstimateSeconds` |
| `number_field`            | Число из поля (`jira_field_id`, напр. `customfield_10016`) как рабочие дни |
| `seconds_field`           | Число из поля как секунды → дни через `working_hours_per_day` |

Если оценка отсутствует или нулевая, используется `min_effort_working_days`.

#### Масштаб шкалы

```yaml
pixels_per_working_day: 36
```

## Запуск

```bash
python app.py
```

Откройте в браузере: [http://127.0.0.1:5000](http://127.0.0.1:5000)

Порт можно задать переменной `PORT` (по умолчанию `5000`).

## API (только чтение)

| Метод | Путь            | Описание |
|-------|-----------------|----------|
| `GET` | `/`             | Страница диаграммы |
| `GET` | `/api/settings` | Настройки для UI (горизонт, цвета, рабочие даты) |
| `GET` | `/api/board`    | Данные доски: строки сотрудников и задачи |

Запись в Jira **не выполняется**: нет эндпоинтов изменения задач, assignee, дат и т.п. Перетаскивание меняет только локальный порядок на экране до обновления страницы / кнопки «Обновить из Jira».

## Как устроена диаграмма

1. Задачи загружаются по JQL и группируются по исполнителю.
2. Внутри строки сортируются по `priorities.order`.
3. Раскладываются слева направо без промежутков: ширина блока ∝ трудозатраты в рабочих днях.
4. Ось X — только рабочие дни горизонта планирования; суббота и воскресенье не отображаются (нулевая длина на шкале).
5. Drag-and-drop меняет строку (сотрудника) и позицию в очереди **только в браузере**.

## Безопасность

- Не коммитьте `.env` с паролями.
- Используйте учётку с минимально необходимыми правами (достаточно чтения задач).
- Приложение предназначено для внутренней сети / локального запуска; отдельной аутентификации веб-UI нет.
